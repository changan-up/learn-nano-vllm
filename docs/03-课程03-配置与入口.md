## 核心概念

### Config：引擎的「物理与调度边界」

`Config` 在初始化阶段被 **`LLMEngine` / `ModelRunner` / `BlockManager`** 消费，决定：

- **能从磁盘加载什么模型**（`model` 目录 + `hf_config`）。
- **单步前向最多算多少 token**（`max_num_batched_tokens`）。
- **最多并发多少条序列**（`max_num_seqs`）。
- **单条序列最长多少**（`max_model_len`，并与 HF 的 `max_position_embeddings` 取 min）。
- **KV 以多大粒度分页**（`kvcache_block_size`）以及 **块数**（`num_kvcache_blocks`，常由显存推算）。
- **张量并行度**（`tensor_parallel_size`）与 **显存利用率**（`gpu_memory_utilization`）。
- **是否禁用 CUDA Graph**（`enforce_eager`）。

可以把 `Config` 理解为：**把「硬件与 SLA」翻译成 Python 数字** 的一层。

### SamplingParams：解码策略（用户侧）

与 **引擎资源** 正交，`SamplingParams` 描述 **如何从 logits 采样下一个 token**：

- **temperature**：控制分布「平」还是「尖」；本项目中 **不允许为 0**（见断言）。
- **max_tokens**：**最多新生成**多少个 token（与 prompt 长度分开理解）。
- **ignore_eos**：若为真，即使遇到 EOS token 也继续生成（用于压测或特殊场景）。

大模型采样 Temperature（温度）通俗详解
一、核心定义
Temperature（简称 T） 是 LLM 文本生成采样环节的超参数，作用是缩放模型输出的 logits 概率分布，用来控制生成结果的：确定性 / 随机性 / 创造性 / 保守程度。
它不改变模型权重、不改变推理计算，只改变「最后选哪个 token」的规则。
二、底层数学原理
模型最后输出一堆 logits（原始打分），要先过 softmax 转成概率。Temperature 的做法：
\(\text{scaled\_logits} = \frac{\text{logits}}{T}\)再对缩放后的 logits 做 softmax 采样。
1. T 越小 → 分布越尖锐
\(T \to 0\)：最高分 token 概率接近 100%
几乎每次都选最确定、最合理的那个词
效果：严谨、死板、几乎不随机、无脑洞

2. T = 1
不缩放，用模型原生概率分布，正常发挥。

3. T 越大 → 分布越平缓
\(T>1\)：把高低分差距抹平，冷门、低概率 token 也有很大机会被选中
效果：脑洞大、自由创作、容易天马行空，太高会乱码、逻辑崩坏


### example.py：最小可运行脚本

典型流程：

1. 指定 **本地 HuggingFace 格式模型目录**（含 `config.json`、权重等）。
2. **`AutoTokenizer`** 加载词表与 chat 模板。
3. 将用户句子包装成 **对话格式字符串**（`apply_chat_template`）。
4. 构造 **`LLM`** 与 **`SamplingParams`**，调用 **`generate`**。
5. 遍历 **`outputs`**，读取 **`output['text']`**（具体键名以仓库为准）。

## 源码解析（带完整源码和逐行注释）

### 1. `config.py`

```python
import os
from dataclasses import dataclass
from transformers import AutoConfig


@dataclass
class Config:
    model: str
    max_num_batched_tokens: int = 16384
    max_num_seqs: int = 512
    max_model_len: int = 4096
    gpu_memory_utilization: float = 0.9
    tensor_parallel_size: int = 1
    enforce_eager: bool = False
    hf_config: AutoConfig | None = None
    eos: int = -1
    kvcache_block_size: int = 256
    num_kvcache_blocks: int = -1

    def __post_init__(self):
        assert os.path.isdir(self.model)
        assert self.kvcache_block_size % 256 == 0
        assert 1 <= self.tensor_parallel_size <= 8
        self.hf_config = AutoConfig.from_pretrained(self.model)
        self.max_model_len = min(self.max_model_len, self.hf_config.max_position_embeddings)
        assert self.max_num_batched_tokens >= self.max_model_len
```

**逐行/逐字段说明**：

| 代码位置 | 含义 | 为什么这么设计 |
|----------|------|----------------|
| `@dataclass` | 自动生成 `__init__` 等 | 配置类字段多，dataclass 减少样板代码 |
| `model: str` | 本地模型目录路径 | 权重与 HF `config` 均从该目录读，**必须是目录** |
| `max_num_batched_tokens` | 单步前向 **批内 token 总数** 上限 | 限制算子输入规模，防止 OOM；需能覆盖 **单条最长序列** |
| `max_num_seqs` | 并发 **序列条数** 上限 | 与连续批处理中「同时服务多少请求」相关 |
| `max_model_len` | 单序列 **最大上下文长度** | 与 KV、块表、调度相关；不能超过模型位置编码上限 |
| `gpu_memory_utilization` | 使用 **多大比例 GPU 显存** 做 KV 等 | 留余量给框架、碎片、临时 tensor |
| `tensor_parallel_size` | **张量并行** 卡数 | 切分权重与张量，值域 1～8 与实现假设一致 |
| `enforce_eager` | **True** 时不用 CUDA Graph，便于调试 | Graph 捕获后难调试；开发期常开 True |
| `hf_config` | HuggingFace `AutoConfig` 对象 | 读取层数、头数、`max_position_embeddings` 等 |
| `eos` | 结束符 token id，默认 -1 表示后续再设 | 与词表、采样结束条件对齐 |
| `kvcache_block_size` | **KV 分页块大小**（token 粒度相关） | PagedAttention 思想；需 **256 的倍数**（与内核/对齐有关） |
| `num_kvcache_blocks` | 块总数，-1 常表示自动算 | 结合显存利用率与块大小推算池大小 |
| `assert os.path.isdir(self.model)` | 路径必须是目录 | 避免把文件路径当模型根目录 |
| `kvcache_block_size % 256 == 0` | 对齐约束 | 简化 Triton/CUDA 内核寻址与块映射 |
| `tensor_parallel_size` 1～8 | 与并行实现匹配 | 防止传入未支持的世界规模 |
| `AutoConfig.from_pretrained` | 拉取 `config.json` | 与 Transformers 生态对齐，少手写超参 |
| `max_model_len = min(...)` | 用户配置与模型能力取 **较小值** | 防止请求超过 **训练/位置编码** 上限 |
| `max_num_batched_tokens >= max_model_len` | **调度可行性**：至少能 **单步吞下一条满长 prompt 的 prefill** | 否则极端情况下无法为超长单请求分配合理 batch |

### 2. `sampling_params.py`

```python
from dataclasses import dataclass


@dataclass
class SamplingParams:
    temperature: float = 1.0
    max_tokens: int = 64
    ignore_eos: bool = False

    def __post_init__(self):
        assert self.temperature > 1e-10, "greedy sampling is not permitted"
```

**逐行说明**：

| 代码 | 含义 | 设计说明 |
|------|------|----------|
| `temperature: float = 1.0` | 默认 **标准 softmax 温度** | 1.0 为「按模型输出分布采样」的常见基准 |
| `max_tokens: int = 64` | 默认最多 **新生成 64 token** | 防止无限生成；业务上常改大 |
| `ignore_eos: bool = False` | 默认 **遇到 EOS 即停** | True 时用于忽略提前结束 |
| `temperature > 1e-10` | **禁止 temperature=0（贪心）** | 贪心需 `argmax` 分支与数值细节；教学项目收窄接口，避免与采样实现分叉 |
| 断言文案 `greedy sampling is not permitted` | 明确告知调用方 | 面试可说：**简化采样路径、与温度采样统一** |

### 3. `example.py`

```python
import os
from nanovllm import LLM, SamplingParams
from transformers import AutoTokenizer


def main():
    path = os.path.expanduser("~/huggingface/Qwen3-0.6B/")
    tokenizer = AutoTokenizer.from_pretrained(path)
    llm = LLM(path, enforce_eager=True, tensor_parallel_size=1)

    sampling_params = SamplingParams(temperature=0.6, max_tokens=256)
    prompts = [
        "introduce yourself",
        "list all prime numbers within 100",
    ]
    prompts = [
        tokenizer.apply_chat_template(
            [{"role": "user", "content": prompt}],
            tokenize=False,
            add_generation_prompt=True,
        )
        for prompt in prompts
    ]
    outputs = llm.generate(prompts, sampling_params)

    for prompt, output in zip(prompts, outputs):
        print("\n")
        print(f"Prompt: {prompt!r}")
        print(f"Completion: {output['text']!r}")


if __name__ == "__main__":
    main()
```

**逐行说明**：

| 行/块 | 作用 |
|--------|------|
| `expanduser("~/...")` | 展开家目录，**跨机器可移植** |
| `AutoTokenizer.from_pretrained(path)` | 从同一模型目录加载 **词表与 special tokens** |
| `LLM(path, enforce_eager=True, tensor_parallel_size=1)` | 构造引擎：`path` 进入 `Config.model`；**eager** 便于调试；**TP=1** 单卡 |
| `SamplingParams(temperature=0.6, max_tokens=256)` | 略低于 1 的温度 = **略更尖锐**的分布；允许更长回答 |
| `prompts` 列表 | 原始用户句 |
| `apply_chat_template(..., tokenize=False, add_generation_prompt=True)` | 按 **Qwen/chat** 格式拼成 **单字符串**（含 `<|im_start|>` 等），**先不转 id**（tokenize 在引擎侧或后续步骤，以仓库为准） |
| `llm.generate(prompts, sampling_params)` | **核心入口**：内部 tokenize + 调度 + 前向 + 采样 |
| `output['text']` | 取 **生成文本**（若你本地版本键名不同，以实际 `llm_engine` 返回为准） |

## 图解（用文字/ASCII 描述）

**配置注入关系**：

```
path (目录)
   |
   +---> AutoTokenizer  （词表、chat 模板）
   |
   +---> Config.model  （HF config + 权重路径）
            |
            v
        LLM / LLMEngine / ModelRunner / BlockManager
```

**example 数据流**：

```
raw prompt
    -> apply_chat_template -> 字符串（带角色标记）
    -> LLM.generate
    -> [输出 dict 列表] -> 打印 text
```

## 面试考点

- **`max_num_batched_tokens` 与 `max_model_len` 的不等式**：保证 **极端单请求 prefill** 仍可被调度（概念题高频）。
- **`kvcache_block_size` 对齐 256**：体现 **内核与内存对齐** 思维。
- **`enforce_eager`**：CUDA Graph 与 **调试/动态形状** 的权衡。
- **`SamplingParams` 禁止 greedy**：**接口简化** 与 **实现统一**。
- **`temperature` 极小值**：数值稳定与 **采样路径唯一**。

## 常见面试题

1. **若 `max_model_len` 设得比 HF 的 `max_position_embeddings` 大会怎样？**  
   答：被 **`min` 截断** 到模型上限，无法突破位置编码能力。

2. **为什么单卡也要写 `tensor_parallel_size=1`？**  
   答：显式表达 **世界规模**，分布式初始化与层实现可统一分支。

3. **`gpu_memory_utilization` 调大可能带来什么风险？**  
   答：KV 池或其它缓冲 **更接近 OOM**；需结合 batch 与并发评估。

4. **`apply_chat_template` 在推理引擎里还是引擎外做更合适？**  
   答：nano-vllm 示例在 **引擎外**；生产 API 常在服务端统一做，**引擎只吃 token id**——能答出 **边界清晰** 即可。

## 小结

- `Config` 定义 **能跑多大 batch、多长序列、多少并发、KV 如何分页**；`__post_init__` 把 **HF 能力** 与 **用户配置** 对齐。
- `SamplingParams` 定义 **解码行为**；通过 **禁止 temperature=0** 收窄采样实现。
- `example.py` 展示 **Tokenizer + LLM + 对话模板** 的标准拼法，是 **跑通与面试叙述** 的第一块素材。
